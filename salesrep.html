Introduction

Overview

This document covers the complete architecture of the Sales Representative App AppSheet application — a mobile-first field CRM built for Enlighten Press sales representatives. The app tracks every aspect of the rep's working day: their location, driving mileage, every client interaction, and daily performance metrics.

The documentation is divided into three parts. Part I explains the three core workflow systems as they appear to the rep. Part II is a complete data dictionary. Part III covers the backend technical implementation: slices, automations, and UI logic.

Technical Note All formulas in this document use AppSheet expression syntax. References to [_THISROW], NOW(), HERE(), and USEREMAIL() are AppSheet system functions evaluated at runtime.
01

Check-in & Check-out System

Core Workflow

The check-in system functions as a digital punch-card. It records when and where the rep starts and ends their working day, and forces collection of daily performance KPIs at the close of each shift.

Daily Workflow
1
Morning — Check In
Rep taps "Check In" on the punch-card screen. The app fires a LINKTOFORM() action that pre-populates Status = "Check in", WorkDate = TODAY(), CheckInTimestamp = NOW(), and CheckInLocation = HERE(). A new row is created in CheckinLog. Rep simultaneously photographs and logs the start odometer reading.
2
During the Day — Client Visits
Rep visits schools and childcares. Each edit to a Leads record triggers the audit bot, appending a row to Rep Visits automatically.
3
Evening — Check Out & KPI Entry
Rep taps "Check Out". A second new row is created in CheckinLog with Status = "Check out". The form dynamically shows the KPI fields (hours, sales, activity counters, end odometer). Rep completes the daily summary. The app calculates total kilometers via a cross-row virtual column formula.
Check In Action Formula
AppSheet Expression · LINKTOFORM 
=LINKTOFORM("CheckinLog_Form",
    "Status",                              "Check in",
    "WorkDate",                            TODAY(),
    "RepName",                             "[Rep Name]",

    "CheckInLocation",                     HERE(),
    "ClaimedHours",                        "0.0",
    "Short summary of how your day went",  "N/A",
    "End of Day Odometer Picture",         "",
    "End of Day Odometer Number",          0
)

AppSheet Expression · LINKTOFORM

    "Status",                       "Check out",

Odometer & Mileage Tracking

Fraud Prevention

The mileage system tracks total kilometers driven daily for reimbursement. It uses AppSheet's Change-type columns to silently lock the GPS coordinates and timestamp the moment a photo is taken, preventing retroactive entry.

Watcher Columns (Fraud Prevention)
Column Type Watches Behaviour
Start of Day Odometer Time Uploaded ChangeTimestamp Start of Day Odometer Picture Locks exact millisecond photo is submitted
Start of Day Odometer Location Uploaded ChangeLocation Start of Day Odometer Picture Locks GPS at moment of photo capture
End of Day Odometer Time Uploaded ChangeTimestamp End of Day Odometer Picture Same — locks evening photo time
End of Day Odometer Location Uploaded ChangeLocation End of Day Odometer Picture Same — locks evening GPS coordinates
Cross-Row Distance Calculation

Because Check In and Check Out generate separate rows in CheckinLog, the daily total must be derived by querying across rows. The virtual column Total KM for the day performs this calculation.

AppSheet Expression · Virtual Column 
=IF(
    ISNOTBLANK([End of Day Odometer Number]),

    [End of Day Odometer Number]
    -
    ANY(
        SELECT(
            CheckinLog[Start of Day Odometer Number],
            AND(
                [WorkDate] = [_THISROW].[WorkDate],
                ISNOTBLANK([Start of Day Odometer Number])
            )
        )
    ),

    ""   /* Returns blank on the Check-In row */
)
How It Works This formula only executes on Check-Out rows (where End of Day Odometer Number is populated). It then SELECT()s the morning row for the same WorkDate, extracts its value with ANY(), and subtracts.
Weekly KM Chart Logic
Slice: Weekly_KM_Slice

Filter: =AND([WorkDate] >= (TODAY() - 7), [WorkDate] <= TODAY())

View type: Histogram — Grouped by WorkDate, Aggregation: MAX :: Total KM for the day. Using MAX ensures the chart ignores the Check-In row (which has a blank/zero value) and plots only the computed Check-Out total.

Rep Visits — Audit Trail System

Immutable Log

The Rep Visits table is the app's immutable audit trail. Reps never interact with it directly. Every time a rep edits any record in the Leads table, a bot fires and appends a new row to Rep Visits, creating a permanent timeline of every action taken.

Leads Table

Master Record

1 row per school or childcare. Always reflects current state. Edited by the rep after each interaction.

Rep Visits Table

Snapshot Log

1 new row per edit event. Never deleted. If a rep visits a school 5 times, there are 5 rows here, each frozen at the moment of that edit.

Deduplication in Charts Editing a single school 4 times in one day creates 4 rows. The Daily Visits Slice uses a MIN(SELECT(...)) pattern to only surface the first log entry per Centre per day, preventing inflated visit counts on the daily chart.

Table: Map Filter

State Table

A single-row state table. Dashboard "Quick Edit" controls are bound to this row. When a rep toggles a switch, this row updates, and all slice formulas referencing it re-evaluate — instantly filtering the map.

Architecture Pattern Update Mode is set to UPDATES_ONLY. The table always has exactly one row (_RowNumber = 1). Slice formulas access it via Map Filter[Column] which returns a single-item List, read with IN() or ANY().
  • Show Schools / Show ChildcaresEnum dropdowns. Controls which location type is rendered on the map.
  • Show UncalledYes/No toggle. When FALSE, hides leads with no prior contact history.
  • Show Open During HolidaysYes/No toggle. Shows/hides holiday-open locations.
  • Show Closed During HolidaysYes/No toggle. Shows/hides holiday-closed locations.
  • Show No Holiday NotesYes/No toggle. Filters locations without any holiday status noted.
  • SelectedDateCalendar picker. Changes which week is displayed on the Weekly Dashboard.
  • Follow Up PickerCalendar picker. Filters map to show all follow-ups scheduled for a specific date.
  • Show Follow Ups TODAYYes/No toggle. Instantly filters map to only today's scheduled follow-up locations.

Slices & Dynamic Filters

AppSheet Expressions
A. Main Map Slice — Unfinished Results

Drives the interactive map. Evaluates the entire Leads table against the current state of the Map Filter row. The formula has two branches: one for when Follow-Up mode is active, one for standard filtering.

AppSheet Expression · Row Filter Condition 
=IF( IN("TRUE", Map Filter[Show Follow ups today]),

    /* TRUE BRANCH: Follow-Up mode — show leads matching the date picker */
    OR(
        AND(
            [Results] = "Follow up",
            ISNOTBLANK([Follow up RC]),
            DATE([Follow up RC]) = ANY(Map Filter[Follow Up Picker])
        ),
        AND(
            NOT([Results] = "Follow up"),
            NOT([Results] = "Successful sale"),
            IF( IN("FALSE", Map Filter[Show Uncalled]),
                ISNOTBLANK([Comments from AS]), TRUE ),
            IF( AND( IN("FALSE", Map Filter[Show No Holiday Notes]),
                     [Results] <> "Open during Holidays",
                     [Results] <> "Closed Due to Holidays" ),
                ISBLANK([Comments from AS]), TRUE ),
            NOT(IN([Results], LIST(
                "No sale","Said no over phone","Call back",
                "Too soon to visit","Couldn't Get To",
                "Said no over email","Successful Call To Be Unmarked"
            ))),
            OR( IN([Type], Map Filter[Show Childcares]),
                IN([Type], Map Filter[Show Schools]) ),
            OR(
                AND(IN("TRUE", Map Filter[Show Open During Holidays]),
                    [Results] = "Open during Holidays"),
                AND(IN("TRUE", Map Filter[Show Closed During Holidays]),
                    [Results] = "Closed Due to Holidays"),
                AND( [Results] <> "Open during Holidays",
                     [Results] <> "Closed Due to Holidays" )
            )
        )
    ),

    /* FALSE BRANCH: Standard filtering, no date override */
    AND(
        NOT([Results] = "Successful sale"),
        IF( IN("FALSE", Map Filter[Show Uncalled]),
            ISNOTBLANK([Comments from AS]), TRUE ),
        IF( AND( IN("FALSE", Map Filter[Show No Holiday Notes]),
                 [Results] <> "Open during Holidays",
                 [Results] <> "Closed Due to Holidays" ),
            ISBLANK([Comments from AS]), TRUE ),
        NOT(IN([Results], LIST(
            "No sale","Said no over phone","Call back",
            "Too soon to visit","Couldn't Get To",
            "Said no over email","Successful Call To Be Unmarked"
        ))),
        OR( IN([Type], Map Filter[Show Childcares]),
            IN([Type], Map Filter[Show Schools]) ),
        OR(
            AND(IN("TRUE", Map Filter[Show Open During Holidays]),
                [Results] = "Open during Holidays"),
            AND(IN("TRUE", Map Filter[Show Closed During Holidays]),
                [Results] = "Closed Due to Holidays"),
            AND( [Results] <> "Open during Holidays",
                 [Results] <> "Closed Due to Holidays" )
        )
    )
)

E. Audit Trail Slices (Rep Visits Table)

Daily Changes Slice

AppSheet Expression 
=AND(
    [Rep Email] = "[rep.email@enlightenpress.com]",
    [Updated] >= (TODAY() - WEEKDAY(TODAY()) + 1),
    [Updated] <  (TODAY() + (7 - WEEKDAY(TODAY())))
)

The date math (TODAY() - WEEKDAY(TODAY()) + 1) calculates the Monday boundary of the current week, creating a rolling 7-day window.

Daily Visits Slice — Deduplication Logic

AppSheet Expression · MIN/SELECT Deduplication 
=AND(
    [Rep Email] = "[rep.email@enlightenpress.com]",
    AND(HOUR(TODAY() - [1st visit RC])/24 <= 7,
        [1st visit RC] <= TODAY()),

    /* Only let the FIRST log entry per Centre per day through */
    [_THISROW].[Timeline] = MIN(
        SELECT(
            Rep Visits[Timeline],
            AND(
                [Centre ID]    = [_THISROW].[Centre ID],
                DATE([1st visit RC]) = DATE([_THISROW].[1st visit RC])
            )
        )
    )
)
Why This Exists Editing a single lead 4 times in one day creates 4 rows. Without this filter, the chart would count 4 visits. MIN(SELECT(...)) finds the oldest timestamp for that Centre + date combination and only allows that one row through the slice.
