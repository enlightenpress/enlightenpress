Introduction & Architecture

Overview

The Australia Leadplanning App is a mobile-first field CRM built for Enlighten Press sales reps operating across Australian states. The app's core design principle is to surface only actionable leads — using complex slice formulas, dynamic map filtering, and cool-down period enforcement so reps never waste time on leads they shouldn't contact yet.

Core Architecture Loop

Single master table. Schools, childcares, sales history.
Control Panel
Map Filter Table
User toggles live here. Slices read these values.
Output
Dynamic Maps
Pins re-render instantly when filters change.
Design Philosophy Rather than building dozens of static views, the app uses a single-row Map Filter state table as a "control panel." All complex slices cross-reference this table using ANY(Map Filter[Column]), so one user input instantly re-filters every downstream view.

Table: Leads

Core Table

The single master table. Every row is a School or Childcare Centre. Permissions: ADDS_AND_UPDATES. Source: Google Sheets worksheet Complete AU.

Identity & Classification Columns
  • ID Text · PK Unique identifier. Prefix system: S# = School, C# = Childcare, H# = Homebase. Drives all type-detection format rules and categorization slices.
  • Type Enum Values: Schools or Childcares. "Allow other values" is enabled. Used in cool-down slice filtering.
  • LatLong LatLong GPS coordinates. The primary column for all Map views. Format rules targeting this column change the map pin's icon and color.
  • Details Centre AS LongText · Sensitive Name and address. Marked as sensitive data. Used as the large header in Detail card views.
  • Primary Phone Number · Sensitive Main contact number. Marked as sensitive.
  • Primary Email Email · Sensitive Main email. Marked as sensitive. Drives the Compose Email action.
  • Comments from AS LongText Historical notes from the master database. Presence or absence of this field drives the "Uncalled" toggle logic in slices — a blank value means the lead has never been contacted.
Visit & Interaction Columns
  • Results Enum Most important field. Last interaction outcome. Drives all filtering, map pin colors, and slice conditions. See status values below.
  • 1st visit RC Date Display name: Last Visit Date. Initial value: TODAY(). Key column for cool-down period math and the "Last Week Results" rolling-7-day slice.
  • Follow Up RC Date Scheduled follow-up date. Initial value: TODAY(). Drives the Calendar view and "Follow Ups" slice.
  • Notes RC LongText Rep's personal notes. Used in cool-down slices to detect "said no over phone" string.
  • Qualifiers Enum · Read-only Lead priority ranking. Values: Low, Medium, New, High, Dead, Best, Potential, Bottom. Drives qualifier-based format rules and the Qualifiers filter on the dashboard.
Sales History Columns
  • # Books PurchasedNumberDisplay name: Total Books Purchased. Drives the numeric-bin filter in dynamic map slices.
  • # Successful SalesNumberLifetime successful sale count. The Repeat Buyers slice requires this to be ≥ 2.
  • # No SalesNumberCount of no-sale outcomes.
  • Date Last Successful SaleDateDrives the Books Sold to before date filter slice — filters who hasn't bought since a chosen cutoff date.
  • Date Last No SaleDateDate of the most recent rejection outcome.
  • Visits Since Last SaleNumberRunning counter of visits without a purchase.
  • Bought in Last 3 VisitsYes/NoConvenience flag for recent purchase history.

Table: Map Filter

State Table

Single-row control table. Permissions: UPDATES_ONLY. Source: Google Sheets worksheet Map Filter. Users never directly see this as a table — they interact with it through Quick Edit controls on the Dashboard view.

  • _RowNumber Number · PK Always row 1. The row number serves as the key for this utility table.
  • Location LatLong · Virtual App formula: =HERE(). Dynamically stores the user's current GPS location. Used to center maps or reference the rep's position.
  • Show Schools / Show Childcares Enum Dropdowns: "Schools" / "None" and "Childcares" / "None". Controls which location type renders on the map. Read in slices via ANY(Map Filter[Show Schools]).
  • Show Uncalled Yes/No When FALSE, the Unfinished Results slice hides leads with blank Comments from AS (never contacted leads).
  • Last Visit Before Date Initial value: TODAY(). The Unfinished Results slice uses this as a date ceiling: only shows leads visited before this date.
  • Last Sold Date Before Date Initial value: TODAY(). Used by the Books Sold to before date filter slice to show customers who haven't bought since this cutoff.
  • Qualifiers EnumList Multi-select. Values: Low, Medium, New, High, Dead, Best, Potential, Bottom. If left blank, slices show all qualifiers.
  • States EnumList Multi-select Australian states: VIC, NSW, WA, etc. Filters which geographic region of leads appears on the map.
  • Books Purchased Filter EnumList Multi-select volume buckets: 0-5, 5-10, 10-20, 20-40, 40+. Dynamic slices translate these string values into numeric boundary checks.

06

Actionable & Time-Based Slices

Call Queues

These slices define who the rep should contact next. They strip out definitive outcomes and surface only leads requiring action.

Slice 4
Last Week Results
READ_ONLY
Row Filter Condition
=AND(
    HOUR(TODAY() - [1st visit RC]) / 24 <= 7,
    [1st visit RC] <= TODAY()
)

Used for the weekly analytics histogram. The division HOUR(...)/24 converts an hour-based duration into days, checking if the first visit happened within the last 7 rolling days. The [1st visit RC] <= TODAY() guard prevents future-dated records from appearing.

Slice 5
Follow Ups
ADDS_AND_UPDATES
Row Filter Condition
=AND(
    NOT(ISBLANK([Follow Up RC])),
    OR(
        CONTAINS([Results], "Books left"),
        [Results] = "Quote sent - call back",
        [Results] = "Follow up"
    )
)

Drives the Calendar view. A lead must satisfy both conditions: it must have a scheduled return date in Follow Up RC and its result must be an actionable pending status.

Slice 6
To be called
ADDS_AND_UPDATES
Row Filter Condition
=AND(
    NOT([Results] = "Successful sale"),
    NOT([Results] = "No sale"),
    NOT([Results] = "Said no over phone"),
    NOT([Results] = "Too soon to visit"),
    OR(
        [Results] = "To be called",
        [Results] = "Call back",
        ISBLANK([Comments from AS])   /* Never contacted */
    )
)

Creates the fresh call queue. Exclusion logic removes any lead with a definitive outcome. The ISBLANK([Comments from AS]) condition captures leads that have never been contacted at all — they have no history in the database yet.

07

Cool-Down Slices

Date Math

Two slices enforce mandatory waiting periods before reps can see or contact leads that previously rejected them over the phone. The math calculates exact months elapsed since the first visit, including sub-month precision.

The Month Calculation Formula (YEAR(TODAY()) - YEAR([1st visit RC])) * 12 + MONTH(TODAY()) - MONTH([1st visit RC]) — multiplies the year difference by 12, then adds the month offset. This avoids the pitfall of naive subtraction failing across year boundaries.
Slice 7
Said no over phone CCs
ADDS_AND_UPDATES

Cool-down period: 4 months — for Childcares.

Row Filter Condition
=AND(
    CONTAINS([Type], "Childcares"),
    CONTAINS([Notes RC], "said no over phone"),

    IF(
        /* Strictly MORE than 4 months ago → show immediately */
        (YEAR(TODAY()) - YEAR([1st visit RC])) * 12
        + MONTH(TODAY()) - MONTH([1st visit RC]) > 4,

        TRUE,

        IF(
            /* Exactly 4 months ago → also check the day */
            (YEAR(TODAY()) - YEAR([1st visit RC])) * 12
            + MONTH(TODAY()) - MONTH([1st visit RC]) = 4,

            DAY(TODAY()) > DAY([1st visit RC]),

            FALSE  /* Less than 4 months → hide */
        )
    )
)

The nested IF handles the edge case where exactly 4 calendar months have passed but the day hasn't been reached yet. E.g., if the 1st visit was Jan 15 and today is May 14, the day check returns FALSE and keeps the lead hidden. On May 15, it returns TRUE.

Slice 8
Said no over phone Schools
ADDS_AND_UPDATES

Identical logic to Slice 7 but targets CONTAINS([Type], "Schools") with a longer cool-down: 5 months.

Row Filter Condition (Diff from Slice 7 shown)
=AND(
    CONTAINS([Type], "Schools"),        /* ← changed */
    CONTAINS([Notes RC], "said no over phone"),
    IF(
        ... > 5,    /* ← threshold is 5 months (not 4) */
        TRUE,
        IF( ... = 5, DAY(TODAY()) > DAY([1st visit RC]), FALSE)
    )
)
Childcares (Slice 7)

Cool-down threshold: 4 months

Schools (Slice 8)

Cool-down threshold: 5 months

Dynamic Map Filter Slices

Cross-Table Logic

The three most complex slices in the app. They cross-reference the Leads table against the user's live inputs stored in Map Filter using ANY(Map Filter[Column]) to read the current single-row state.

Slice 9
Unfinished Results
UPDATES_ONLY

The primary map engine. Evaluates every lead against all active dashboard filters simultaneously.

Row Filter Condition (Annotated)
=AND(

    /* 1. Date filter: only leads visited before the chosen cutoff */
    ANY(Map Filter[Last Visit Before]) > [1st visit RC],

    /* 2. Exclusion: strip definitive "closed" outcomes */
    NOT([Results] = "Successful sale"),
    NOT([Results] = "No sale"),
    NOT([Results] = "Closed Due to Holidays"),

    /* 3. Uncalled toggle: if user hid uncalled, require prior history */
    IF(
        CONTAINS(Map Filter[Show Uncalled], "FALSE"),
        ISNOTBLANK([Comments from AS]),
        TRUE
    ),

    /* 4. Type toggle: show if type matches user's selected category */
    OR(
        IN([Type], Map Filter[Show Childcares]),
        IN([Type], Map Filter[Show Schools])
    ),

    /* 5. Qualifiers multi-select: blank = show all */
    OR(
        ISBLANK(ANY(Map Filter[Qualifiers])),
        CONTAINS(TEXT(ANY(Map Filter[Qualifiers])), [Qualifiers])
    ),

    /* 6. States multi-select: blank = show all states */
    OR(
        ISBLANK(ANY(Map Filter[States])),
        CONTAINS(TEXT(ANY(Map Filter[States])), [States])
    ),

    /* 7. Books purchased numeric bin check */
    AND(
        IN("0-5",   Map Filter[Books Purchased Filter]) AND [# Books Purchased] >= 0  AND [# Books Purchased] <= 5,
        ...  /* OR for each bucket: 5-10, 10-20, 20-40, 40+ */
    )
)
Multi-Select Pattern OR(ISBLANK(ANY(Map Filter[Qualifiers])), CONTAINS(...)) — if the user leaves the filter blank, the ISBLANK short-circuits to TRUE and shows everything. If they select specific values, the CONTAINS check filters to only matching leads. This pattern repeats for both Qualifiers and States.
Slice 10
Repeat Buyers
UPDATES_ONLY

Identical to Unfinished Results with one additional condition prepended at the top of the AND() block:

Additional Condition (Prepended)
[# Successful Sales] >= 2

Strictly isolates customers who have made at least two separate purchases. All the same Map Filter controls (type, qualifier, state, books bin) still apply.

Slice 11
Books Sold to before date filter
ADDS_AND_UPDATES

Same dynamic filtering block as Unfinished Results but replaces the first-visit date check with a last-sale date check:

Modified Date Filter (vs. Slice 9)
/* Instead of: ANY(Map Filter[Last Visit Before]) > [1st visit RC] */
AND(
    ISNOTBLANK([Date Last Successful Sale]),
    ANY(Map Filter[Last Sold Date Before]) > [Date Last Successful Sale]
)

Only shows customers who haven't bought anything since the rep's selected date cutoff — a targeted re-engagement list for lapsed buyers.
